## Quantity boundary

The complete component-entry distribution is:

| Component quantity | Entries |
| --- | ---: |
| 1 | 23,891 |
| 2 | 399 |
| 4 | 4 |
| 8 | 401 |

The 406 rejected parent SKUs contain one or more `x2`, `x4`, or `x8` relationships.
They produce 804 `UNSUPPORTED_COMPONENT_QUANTITY` findings because a single Bundle
may contain multiple repeated-quantity components.

The current pre-built import contract, runtime projection, and Function result all
emit component quantity `1`. The preflight therefore fails closed instead of
flattening, duplicating, or changing quantities. Supporting repeated quantities
requires a separately reviewed V5.5-or-newer architecture decision and runtime,
price-allocation, Checkout, Order, and inventory tests.

## Local implementation

- `prebuilt-bundle-import.bundles-app-catalog.js` parses relationships, removes only
  exact duplicate parent relationships, resolves Shopify identities, and classifies
  conflicts without producing target mappings or writes.
- `preflight-bundles-app-catalog.mjs` reads the Excel workbook and Variant CSV,
  creates a deterministic report, rejects Shopify mutation flags, and can write only
  a new local JSON report using no-overwrite semantics.
- `npm run preflight:bundles-app-catalog` is the reusable operator command.
